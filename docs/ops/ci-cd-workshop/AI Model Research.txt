PERPLEXITY RESEARCH PROMPT (List of Models)
---------------------------------------
Research the following AI models:

[LIST_OF_MODELS]

For every model provided in the list, provide answers to the following questions so they can be accurately compared:

**Lifecycle & Basic Information**
* When was each specific model version officially released?
* Are there published deprecation dates or end-of-life warnings for any of these model versions?
* Are these strictly API-only models, or are the weights open for any of them?

**Context Window & Pricing**
* What is the absolute maximum input token limit for a single request for each model?
* What is the absolute maximum output token limit for a single response for each model?
* What is the standard cost per 1 million input tokens for each model?
* What is the standard cost per 1 million output tokens for each model?
* Which of these models offer discounted pricing for cached input tokens, and what is that specific cost?

**Technical Integration & Capabilities**
* Which of these models natively support tool calling and function calling?
* Which of these models support a strict JSON output mode or structured outputs?
* Which of these models support system-level prompt caching?
* What native modalities does each model support (e.g., text-only, vision, audio, document parsing)?
* What specific development or operational tasks is each model known to excel at, particularly in comparison to the others on this list?
* What specific tasks or reasoning types is each model known to struggle with, particularly in comparison to the others on this list?

**Performance Metrics**
* What is the generally observed Time to First Token (TTFT) for each model?
* What is the generally observed generation speed (Tokens Per Second) for each model?
* How does each model rank currently on major benchmarks like the LMSYS Chatbot Arena or HumanEval, and how do they compare to each other?

**Behavior & Enterprise Compliance**
* How strictly does each model adhere to complex system prompts during long context interactions?
* What are the known or common refusal triggers for each model, particularly regarding code generation, automation, or technical queries?
* Do the providers' standard API agreements for these specific models state that they will use customer API data to train future models?

We are refactoring our agent framework to min-max token usage, eliminate codebase drift, and improve development efficiency. We are moving away from brute-forcing tasks with the most expensive model.

Core Directives

1. Update Agent Skills & Subagents

User-Facing Skills: Refactor all skills that produce terminal output to the user. They must now generate and recommend contextual follow-up actions for the agents.

Planning Subagents: Update all skills and subagents responsible for creating execution plans to [Insert missing instruction here - e.g., "also append validation follow-ups to their plans"].

Skill Focus: Prioritize stability by restricting current development to DevOps and ITIL-focused skills.

2. Implement Dynamic Model Routing

Deprecate the hardcoded use of the single most expensive model across the board.

Implement a routing logic that selects the appropriate model based on the specific skill being executed and the execution environment (Cloud vs. Local Workstation).

3. Establish a Single Source of Truth (SSOT) for Models

Take the attached model research and ingest it into the repository as a centralized configuration file or documentation standard.

Refactor all existing skills to reference this central repository for model parameters and capabilities rather than holding their own local definitions.
